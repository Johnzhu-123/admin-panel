feat(admin): v22 — 服务清单与桌面端 1:1 对齐（5 tabs / 6 categories / auto-probe / vision 独立）

BACKGROUND:
v21 之前 admin-panel 把每个 category 切成 2-3 个子任务（如 multimodal=音频理解+通用多模态+视觉理解、image=generation+understanding+edit），
但桌面端「API 设置」面板每个 tab 只调用 1 个 renderCategoryCatalogEditor（image tab 例外，调 2 次：image+vision）。
这导致：(1) 管理员要在 admin 入库 3 次但默认预设只能选 1 个；(2) 桌面端按 category 单维度读取时，模型下拉缺料/混杂。

CHANGES:

   - SUBTASK_PRESETS 全部简化为单 default 子任务（不再有 generation/understanding/edit/audio_understanding 等多桶）
   - 新增 legacy migration：旧的 3-子任务数据按 baseUrl/apiKey/model 信息密度自动收敛到 default
   - 特殊：image:understanding 老数据自动迁移到 vision:default 兜底
2. components/admin/service-catalog-management.tsx — 完全重写（946 行）
   - 5 个 Tabs：多模态 / 文本 / 图像 / 配音 TTS / 视频
   - 「图像 AI」tab 横向并列两个独立表单（图像生成 image + 图像理解 vision）

     + 「分类模型」输入框（datalist 候选 = 上游 /models 探测结果）+ 加入分类 + 全部加入 + 一键移除
